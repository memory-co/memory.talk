# 用户故事：统一消息管理（Subject）

## 背景

**用户场景**：
用户希望将来自不同平台（Claude Code、ChatGPT、Cursor 等）的对话记录统一收集到本地进行管理和检索。

**当前问题**：
- 不同平台的消息来源标识混乱（有的叫 user，有的叫 human，有的叫 assistant）
- 无法区分不同 AI 模型生成的消息（如 Claude 3.5 Sonnet vs GPT-4）
- 无法区分人类用户和 AI 助手
- 无法区分 AI 本体和 AI 调用的工具（Tool）

---

## 用户故事

### 故事 1：作为用户，我希望将 Claude Code 的对话历史导入并正确识别消息来源

**完整流程**：

1. 用户在 Claude Code 中完成一天的工作
2. 用户运行 Claude Code 导出工具（export_sessions.py）
3. 工具扫描本地项目目录，读取对话历史文件
4. 工具将对话数据发送到 memory-talk 服务器
5. 服务器自动分析每条消息：
   - 用户输入 → 标记为「人类用户」
   - AI 回复（有 model 信息）→ 标记为对应的 AI 模型
   - AI 调用工具 → 标记为「工具」
6. 用户打开 dashboard，在对话详情页查看导入的对话
7. 用户可以看到每条消息左侧有不同的标识，区分人类、AI、工具

**用户价值**：
- 无需手动标记，系统自动区分人类和 AI
- 可以按来源筛选对话历史
- 支持后续按模型维度分析对话

---

### 故事 2：作为用户，我希望自定义消息来源的显示名称和属性

**完整流程**：

1. 用户在 dashboard 的「来源管理」页面
2. 用户点击「新建来源」按钮
3. 用户填写来源信息：
   - ID：claude-project-alpha
   - 名称：项目专属 Claude
   - 颜色：#FF9900
4. 用户保存，新的来源出现在列表中
5. 用户修改已有来源的显示名称或颜色
6. 用户删除不再需要的来源，确认后该来源消失
7. 用户注意到被删除来源关联的消息仍然存在，只是显示为「未分类」

**用户价值**：
- 为不同项目配置专属的 AI 身份
- 用颜色区分不同来源的消息
- 灵活适应各种工作流

---

### 故事 3：作为用户，我希望系统默认预置常用的消息来源

**完整流程**：

1. 用户首次安装并启动 memory-talk 服务
2. 系统自动创建两个默认来源：
   - 人类用户（用于标记所有人类输入）
   - AI 助手（用于标记未识别模型的 AI 回复）
3. 用户无需任何配置，直接开始导入对话
4. 用户后续启动服务时，系统检测到来源已存在，不再重复创建

**用户价值**：
- 开箱即用，无需额外配置
- 确保数据导入时能正确匹配
